Codex UI Linux Port
Pipeline Packages Publication

About

Unofficial Linux packaging automation for Codex UI.

Codex UI Linux Port exists to make frequent Codex UI updates repeatable on Linux systems without committing upstream binaries or maintainer-local runtime data to git.

Scope

  • Resolve current upstream source archive release metadata.
  • Rebuild Linux native modules.
  • Apply Linux desktop integration patches.
  • Generate Arch/CachyOS, Debian, and RPM packages.
  • Validate manifest, checksums, and privacy boundary.

Non-Goals

  • No official OpenAI support claim.
  • No extracted upstream app bundles committed to git.
  • No Codex chats, profiles, sessions, runtime state, tokens, or private keys.
  • No AUR publication before redistribution and trademark review.

Public automation. Unofficial releases.

Release artifacts are generated by GitHub Actions from upstream Codex UI release archives. Project wording, metadata, and assets must remain clearly unofficial.

  1. Review upstream redistribution and trademark constraints.
  2. Keep release assets generated by CI, not committed to git.
  3. Run scripts/privacy-audit from a clean checkout.
  4. Verify docs and package metadata describe the project as unofficial automation.
  5. Update AUR metadata only after release asset policy is decided.

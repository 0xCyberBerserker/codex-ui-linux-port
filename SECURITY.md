# Security Policy

## Supported Versions

Security fixes target the current `main` branch and latest release. Older generated packages are not maintained.

## Reporting A Vulnerability

Do not disclose suspected vulnerabilities, credentials, tokens, private paths, or runtime data in a public issue.

Use GitHub's private vulnerability reporting or a private security advisory for this repository. Include affected version, reproduction steps, impact, and any proposed mitigation.

## Scope

Reports may cover repository automation, release workflows, package integrity, update verification, privilege handling, privacy controls, and Linux-specific patches.

Vulnerabilities in upstream Codex UI or OpenAI services should be reported through OpenAI's official security channels.

See [docs/security.md](docs/security.md) for repository privacy and release controls.

# About Codex UI Linux Port

Codex UI Linux Port is an unofficial packaging automation project for Linux systems.

The project exists to make frequent Codex UI updates repeatable on Linux without committing upstream binaries or runtime data to git. It uses GitHub Actions as the release authority, validates generated packages, and keeps redistribution boundaries explicit.

## Operational Rationale

This project covers a practical need from daily engineering work. Codex UI is part of a regular Linux workstation workflow, and frequent upstream updates made manual repackaging wasteful and fragile.

The automation turns unavoidable waiting periods such as long builds, dependency installs, and CI feedback loops into useful maintenance time. It improves workstation reliability and day-to-day engineering throughput without storing private runtime data, credentials, chats, or local project material.

## Scope

The project provides:

- automated source archive resolution from upstream release metadata
- Linux native module rebuilds
- Linux desktop integration patches
- Arch/CachyOS package output plus experimental Debian and RPM outputs
- checksum and manifest validation
- a host-aware updater command: `codexui-update`
- future AUR metadata

## Non-Goals

The project does not:

- claim official OpenAI support
- redistribute source ownership
- commit extracted upstream application bundles to git
- store Codex chats, profiles, sessions, runtime state, tokens, or private local data
- publish AUR packages before redistribution and trademark review

## Publication Model

The repository is public automation. Release assets are generated by GitHub Actions and must remain clearly marked as unofficial.
